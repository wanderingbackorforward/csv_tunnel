# 面试回答：和传统 BI 的区别

**问题：你这个项目和传统 BI、报表系统相比，本质区别是什么？是不是只是规则 + 图表？**

---

我觉得这个问题问得很好，因为表面上确实有点像——都是处理数据、都有阈值判断。

传统 BI 能做的事情我承认：它可以画折线图、做报表、设阈值告警，这些都是有价值的。如果你只是想看"某个时刻转矩是多少"，BI 完全够用。

但这个项目多做了几件事，我觉得这才是本质差异：

第一，它不是告警，是**自动发现问题**。BI 的阈值告警是"这个点超了"，我的工具是"这段时间内有一个持续的异常事件，从几点到几点，持续了多少秒"。它会自动把连续命中的点压成一个事件，而不是给你一堆孤立的告警点。

第二，它会**自动组织证据**。每个事件会附带：哪些信号异常、均值是多少、峰值是多少、变化趋势怎样。这些不需要人去翻图表，工具直接提取出来。

第三，它会**生成工程人员可读的解释**。不是"转矩超阈值"，而是"该时段刀盘转矩持续偏高，均值 3,447 kNm，疑似地层阻力增大，建议复核地质记录"。

第四，也是我后来加的，它有**工况状态层**。系统先识别当前是停机、低负载、正常推进还是重载推进，再去解释异常。同样的液压波动，在切换泵组时和在正常推进中，含义是不一样的。BI 不会做这个区分。

所以我不会说它是"强 AI agent"，它没有那么复杂。但它比 BI 多走了一步：从"展示数据让人判断"，到"自动发现问题、组织证据、给出初步解释"。更像一个智能诊断工具，而不是一个报表系统。
