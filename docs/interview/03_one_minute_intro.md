# 1 分钟面试口述项目介绍

这个项目的起点是一个很现实的约束：对方的设备系统只能持续导出 CSV 文件，没有办法直接接实时数据流。所以我没有去做什么流处理平台，而是做了一个本地 CLI 工具，专门处理这些 CSV。

工具的核心链路是这样的：先做字段识别和映射，因为不同批次的 CSV 列名可能不一样；然后做清洗，处理缺失值、重复时间戳、尖峰噪声；再提取时序特征，比如滚动均值、贯入度、转矩速度比；然后用规则检测异常点，把连续命中的点合并成事件段，再为每个事件提取证据、生成模板解释，最后导出 JSON、Markdown 或 events.csv。

后来我发现一个问题：同样是"推进速度偏低"，如果机器当时在停机，和在正常推进中，含义完全不同。所以我加了一个工况状态识别层，先把每行数据分类为停机、低负载、正常推进、重载推进，再把这个状态汇总到事件解释里。这样输出的结论就不只是"速度低于阈值"，而是"该事件主要发生在重载推进状态下，疑似地层阻力增大"。

我觉得这个项目最有价值的地方不是代码量，而是在现实约束下把一条可用的链路做稳了——在 8.6 万行真实数据上跑通，处理过缺列、混合时间格式、短时尖峰误报这些真实问题，不是只跑了一个 sample demo。
