# 简历项目表达

## 一句话定位

在"对方系统只能导出 CSV"的现实约束下，为盾构/TBM 施工数据构建了一套外挂式智能诊断工具：规则检测 + 工况语义 + LLM 归纳 + 两阶段批量复核，把原始时序 CSV 转化为工程师可直接使用的事件报告。

---

## 简历项目标题建议

**盾构/TBM 时序数据智能诊断系统（Python CLI）**

或更简洁版：

**TBM 施工数据异常检测与 AI 复核工具**

---

## 简历 Bullet（4 条）

- 设计并实现完整时序诊断链路（字段映射 → 清洗 → 特征提取 → 规则检测 → 事件分段 → 证据提取 → 模板解释 → 多格式导出），各模块单一职责，检测逻辑与解释逻辑严格分离；在真实 TBM 数据（单文件 8.6 万行）上验证，对缺列、重复时间戳、损坏文件均不崩溃。

- 引入工况状态识别层（停机 / 低负载 / 正常推进 / 重载推进）+ 事件语义重分类层（semantic layer），将"低效掘进"事件按主导工况拆分为"停机片段"与"推进中低效掘进"，消除了大量停机数据被误报为推进效率问题的语义混淆，使 AI 总结结论从"推进参数偏差"修正为"停机时间过长"。

- 构建两阶段批量分析流程：`scan` 子命令对全量历史文件（数十个 XLS）并行规则诊断并生成 `scan_index.csv`；`review` 子命令按风险分筛出 Top N 文件调用 LLM 深度复核，生成含语义事件分布统计和跨文件核心问题判断的 `review_summary.md`，将 AI 调用成本集中在真正高风险文件上。

- 以零侵入方式在规则检测基线上叠加 OpenAI-compatible tool-using agent 层（适配 MiniMax API），agent 通过工具调用驱动完整检测链路，支持多轮推理；LLM 总结层和 agent 层均设五级降级（无 SDK / 无 key / 无事件 / 超时 / JSON 解析失败），任意失败不影响规则主链路输出。

## 使用场景说明

以上 bullet 适合放在"项目经历"栏，面向数据工程、后端、AI 应用方向的岗位。
