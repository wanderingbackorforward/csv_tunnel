# 简历项目经历

## 项目标题

盾构/TBM 设备运行数据诊断工具（Python CLI）

---

## 简历 Bullet

- 设计并实现完整的时序数据处理链路（字段映射 → 清洗 → 特征提取 → 规则检测 → 事件分段 → 证据提取 → 模板解释），各模块单一职责，接口通过 dataclass 传递，检测逻辑与解释逻辑严格分离。

- 引入工况状态识别层，对每行数据分类为停机/低负载/正常推进/重载推进，将事件窗口内主导状态注入解释输出，使异常报告从"数值超阈值"升级为带工况语义的结论。

- 针对真实数据中的短时尖峰误报问题，通过在分段层引入 `min_event_points` 参数过滤 3 点以下孤立事件，在不影响长时真实异常检出的前提下消除噪声触发。

- 在 8.6 万行真实 TBM 数据上完成验证：正常段 0 误报，明显异常段可检出；工具对缺列、重复时间戳、混合时间格式、损坏文件均不崩溃，支持 watch 目录自动处理与 YAML 外部配置。
